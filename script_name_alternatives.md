# Alternatives au nom "web_to_md"

## Option 1: `webmark` ⭐ (Recommandé)
**Avantages:**
- Court et mémorable
- Évoque clairement "web" + "markdown"
- Facile à taper
- Professionnel

**Utilisation:**
```bash
python webmark.py https://example.com/article
webmark --crawl https://example.com/blog/
```

## Option 2: `mdscrap`
**Avantages:**
- Clair sur la fonction (scraping → markdown)
- Court (7 caractères)
- Évite confusion avec "web2md" (trop générique)

**Utilisation:**
```bash
python mdscrap.py https://example.com/article
mdscrap --sitemap https://example.com/sitemap.xml
```

## Option 3: `harvest`
**Avantages:**
- Métaphore élégante (récolter du contenu web)
- Court et unique
- International (pas de "2" ou "to")
- Évoque l'idée de crawling/collecte

**Utilisation:**
```bash
python harvest.py https://example.com/blog/
harvest --crawl --max-depth 2 https://example.com/docs/
```
